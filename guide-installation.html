ÉTAPE 1 — Configurer les profils Hotspot

TERMINAL MIKROTIK

Ouvre Winbox → Terminal et colle ces commandes :

# ── Créer les 10 profils de forfaits Cyber Kim ── /ip hotspot user profile add name="Flash" session-timeout=00:15:00 rate-limit="2M/2M" shared-users=1 add name="Express" session-timeout=00:30:00 rate-limit="2M/2M" shared-users=1 add name="Starter" session-timeout=00:45:00 rate-limit="2M/2M" shared-users=1 add name="Standard" session-timeout=01:00:00 rate-limit="2M/2M" shared-users=1 add name="Confort" session-timeout=01:30:00 rate-limit="2M/2M" shared-users=1 add name="Plus" session-timeout=02:00:00 rate-limit="2M/2M" shared-users=1 add name="Max" session-timeout=02:30:00 rate-limit="2M/2M" shared-users=1 add name="Premium" session-timeout=03:00:00 rate-limit="2M/2M" shared-users=1 add name="Pro" session-timeout=03:30:00 rate-limit="2M/2M" shared-users=1 add name="Ultra" session-timeout=04:00:00 rate-limit="2M/2M" shared-users=1
# ── Persistance des sessions (protection anti-coupure courant) ── /ip hotspot profile set [find] keepalive-timeout=00:02:00 /ip hotspot set [find] idle-timeout=00:05:00
✅ L'onduleur branché sur le MikroTik protège contre les micro-coupures. Ces paramètres permettent une reconnexion automatique si le client perd brièvement le signal.

ÉTAPE 2 — Uploader le portail captif personnalisé

FICHIERS À UPLOADER

Le portail MikroTik lit ses fichiers HTML depuis :

ÉTAPE 3 — Configurer le profil Hotspot

WINBOX → IP → HOTSPOT → SERVER PROFILES
# Dans le terminal MikroTik : /ip hotspot profile set [find] login-by=http-chap \ html-directory=hotspot \ http-cookie-lifetime=3d \ use-radius=no
✅ login-by=http-chap : authentification sécurisée via le formulaire HTML personnalisé Cyber Kim.

ÉTAPE 4 — Nommer le WiFi (SSID)

WINBOX → WIRELESS
# Renommer le réseau WiFi visible par les clients : /interface wireless set [find] ssid="CYBER KIM" # Optionnel : réseau séparé pour les postes internes : /interface wireless set wlan1 ssid="CYBER KIM - WiFi"

ÉTAPE 5 — Activer l'API REST MikroTik NOUVEAU

DASHBOARD LIVE — PRÉREQUIS

L'onglet 🔴 Live MikroTik de l'app Cyber Kim interroge directement le routeur via l'API REST. Voici comment l'activer :

5.1 — Activer le service web (API REST)

# Activer le service HTTP sur port 80 /ip service enable www /ip service set www port=80 # Activer l'API sur port 8728 (optionnel, pour outils tiers) /ip service enable api /ip service set api port=8728

5.2 — Créer un utilisateur dédié (lecture seule)

# Créer le compte "dashboard" en lecture seule (sécurité) /user add name=dashboard password=CyberKim2025! group=read # Vérifier que l'utilisateur est créé : /user print

5.3 — Autoriser l'accès depuis le PC gérant uniquement

# Remplace 192.168.17.X par l'IP fixe du PC gérant /ip service set www address=192.168.17.0/24 # Vérifier que l'API répond — depuis le navigateur du PC : # http://192.168.17.1/rest/system/identity # → Doit retourner : {"name":"MikroTik"}

5.4 — Activer CORS (pour que l'app puisse interroger l'API)

# Autoriser les requêtes depuis le navigateur local /ip service set www-ssl certificate=none # Si le navigateur bloque (erreur CORS), utilise cette URL # dans l'app à la place de 192.168.17.1 : # http://192.168.17.1/rest/ip/hotspot/active
⚠️ Si tu vois une erreur CORS dans le navigateur : c'est une limitation de sécurité du navigateur. Solution : ouvre l'app Cyber Kim depuis file:/// (fichier local) ou utilise Firefox qui est plus permissif avec les requêtes locales.
✅ Test rapide : depuis le PC gérant, ouvre ce lien dans Firefox : http://192.168.17.1/rest/ip/hotspot/active — si tu vois du JSON, l'API fonctionne !

ÉTAPE 6 — Utiliser le Dashboard Live NOUVEAU

APP CYBER KIM → ONGLET 🔴 LIVE

6.1 — Ouvrir l'app

  • Ouvre cyber-kim-jetons.html dans Firefox (recommandé)
  • Clique sur l'onglet 🔴 Live MikroTik

6.2 — Configurer la connexion

  • IP MikroTik : 192.168.17.1
  • Utilisateur : dashboard
  • Mot de passe : celui défini à l'étape 5.2
  • Clique ⚡ Connecter

6.3 — Ce que tu vois en temps réel

  • 🟢 Sessions actives — nombre de clients connectés en ce moment
  • 🎫 Jetons utilisés / restants — état des comptes sur MikroTik
  • 💰 Recettes du jour — calculées depuis les jetons locaux
  • 🌐 Trafic réseau — upload / download avec barres animées
  • 👥 Tableau des sessions — MAC, IP, forfait, temps restant, données consommées
  • 🚨 Alertes automatiques — connexions, déconnexions, erreurs
🔄 Le dashboard s'actualise automatiquement toutes les 10 secondes sans intervention.

6.4 — Déconnecter un client manuellement

Dans le tableau des sessions, clique sur le bouton DÉCO en face du client → il est immédiatement déconnecté du réseau.

# Équivalent MikroTik en ligne de commande : /ip hotspot active remove [find where mac-address="XX:XX:XX:XX:XX:XX"]
